### 开发经验总结

#### 1. 状态管理设计
- 函数级别的属性（如 `showDerivative`）应放在函数对象内
- 避免全局开关导致所有函数同时生效
- 使用 `toggleFunctionXxx(id)` 模式操作单个函数

#### 2. 数值计算精度
- 中心差分法求导：`(f(x+h) - f(x-h)) / 2h`
- 步长 `h = 1e-5` 在精度和稳定性间取得平衡
- 需要处理 NaN 和 Infinity 情况

#### 3. Canvas 导出
- 使用 `canvas.toDataURL('image/png')` 转换
- 通过 store 存储 canvas 引用，避免跨组件传递
- 创建临时 `<a>` 元素触发下载

#### 4. UI 按钮设计原则
- 图标要有明确语义（实心/空心表示状态）
- 下拉菜单必须有方向指示
- 激活状态要有视觉反馈（颜色、背景）
- 悬停时显示，避免界面杂乱

#### 5. 采样算法优化
- 用户设置的采样数应真正生效
- 视口范围大时自动加密，但基于用户基数
- 公式：`dynamicCount = userCount × densityFactor`
